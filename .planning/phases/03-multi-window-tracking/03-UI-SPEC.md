## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon button padding, action button padding (`padding: 3px` rounds to 4px), swatch gap |
| sm | 8px | Workspace item vertical padding, modal action gap, color picker gap |
| md | 16px | Popup horizontal padding (14px used → treat as md context), modal padding bottom |
| lg | 24px | — reserved for future layout gaps |
| xl | 32px | — reserved |
| 2xl | 48px | — reserved |
| 3xl | 64px | — reserved |

**Source:** Derived from `src/popup/popup.css` measurements. The popup uses 14px horizontal padding (closest to md) and 8px vertical padding consistently.

Exceptions:
- Popup fixed width: 300px (design constraint, not a spacing token)
- Popup max-height: 500px (design constraint)
- Workspace item left border (active indicator): 3px width, inset 4px top/bottom — stays as-is (not a spacing token)
- Color swatch size: 26px × 26px — component-specific, not a layout spacing token
- In-use indicator icon: 14px × 14px — matches existing `makeSvgIcon` dimensions (width="14" height="14")
- Unassigned banner: 8px vertical padding, 14px horizontal padding — matches existing item rhythm

## Component Inventory

New and modified UI components for this phase. All built via `document.createElement` / `createElementNS` — no framework components.

### 1. In-Use Indicator Icon (new)

- Element: inline SVG via `makeSvgIcon()` (existing helper)
- Placement: appended inside `.ws-info` after the name, or as a sibling to `.ws-actions` — to the right of workspace name, before the action buttons
- SVG path: two overlapping rectangles/windows glyph — use `'M2 4h10v9H2zM5 2h9v9'` (window-on-window) with `stroke="currentColor" stroke-width="1.3" stroke-linecap="round" stroke-linejoin="round"`
- Size: 12px × 12px (slightly smaller than edit/delete icons at 14px to visually subordinate it)
- Color: `#6c7086` (muted, not alarming — signals state, not action)
- Visibility: always visible when `isInUse === true`, hidden otherwise (no opacity animation needed — it is a state indicator, not a hover reveal)
- Tooltip (`title` attribute): "Active in another window"

**Source:** CONTEXT.md D-05 (subtle icon indicator, same list, not dimmed); RESEARCH.md Claude's Discretion — icon choice.

### 2. Workspace Item — In-Use State

- Class modifier: `workspace-item--in-use` (added alongside existing classes; does NOT replace active class)
- Visual treatment: no dimming, no separate grouping (D-05 is explicit). The only visual change is the presence of the in-use indicator icon
- Click behavior: fires `focusWindow` message to background, then calls `window.close()` to dismiss popup (D-06)
- Cursor: `pointer` (unchanged — clicking is the action to focus the other window)

### 3. Workspace Item — Current Window Active State

- Existing class: `.workspace-item.active` already provides `#2a2a42` background and `var(--ws-color)` left border
- Phase 3 change: the active indicator is now per-window. The `isActive` check in `renderList()` uses `activeWorkspaceId` returned from `getState` (which is now `windowMap[currentWindowId]` for the popup's window)
- No new CSS class needed — the existing `.active` class conveys this correctly

**Source:** CONTEXT.md D-07 (visual treatment like background or border, not just text); existing `.workspace-item.active` CSS.

### 4. Unassigned Window Banner (new)

- Element: `<div class="ws-unassigned-banner">` inserted above the workspace list when `activeWorkspaceId === null`
- Content: icon (open window glyph, 14px) + text "No workspace assigned"
- Background: `#252536`
- Text color: `#6c7086` (muted)
- Font: 12px, weight 400
- Padding: 8px 14px (matches workspace item horizontal padding)
- Border-bottom: `1px solid #2e2e3e` (matches header border)
- Visibility: only rendered when popup window is unassigned (D-08)

**Source:** CONTEXT.md D-09 (unassigned window shows full list + create + move options).

### 5. "Assign Here" Inline Action (new)

- Element: additional button in `.ws-actions` area for each workspace row, visible only when the current window is unassigned
- Label: "Assign" (text button, not icon-only — action is not familiar enough to be icon-only)
- Font: 11px, weight 500
- Color at rest: `#89b4fa` (accent — this IS an assignable action, the primary interaction for unassigned windows)
- Background: none (matches `.ws-actions button` base style)
- Hover: `background: #2e2e4e`, color `#cdd6f4`
- Visibility: hidden when window is assigned; shown only in unassigned window state — toggled via a parent class on the list container (e.g., `workspace-list--unassigned`) to avoid per-item logic
- Click behavior: sends `assignWorkspace` message (saves current window tabs into workspace, activates it), then re-renders list
- Placement: leftmost button in `.ws-actions` (before edit and delete buttons)

**Source:** CONTEXT.md D-09; RESEARCH.md Open Question 3 recommendation (inline action per workspace item, avoids modal).

### 6. Per-Window Badge

- Text: workspace name first character, uppercase (`workspace.name.charAt(0).toUpperCase()`)
- Background: `sanitizeColor(workspace.color)` — the workspace's hex color
- Unassigned text: `'?'`
- Unassigned background: `#888888`
- API: `browser.action.setBadgeText({ text, windowId })` and `browser.action.setBadgeBackgroundColor({ color, windowId })`

**Source:** RESEARCH.md Pattern 4; CONTEXT.md Claude's Discretion — badge for unassigned windows; RESEARCH.md recommendation: `'?'` with `#888888`.

## Interaction Contract

### Clicking an in-use workspace (D-06)

1. User clicks a `.workspace-item--in-use` row
2. Popup sends `{ action: 'focusWindow', windowId: owningWindowId }` to background
3. Background calls `browser.windows.update(windowId, { focused: true })`
4. Popup calls `window.close()` immediately after receiving success response
5. No confirmation dialog — focusing another window is non-destructive

### Clicking an available workspace (existing, unchanged)

1. User clicks a `.workspace-item` row that is neither active nor in-use
2. Popup sends `{ action: 'switchWorkspace', workspaceId, windowId: currentWindowId }`
3. All list items set to `opacity: 0.5` during transition (existing pattern)
4. On response, `renderList()` is called

### Assigning a workspace to an unassigned window (new)

1. User clicks "Assign" inline button on a workspace row
2. Popup sends `{ action: 'assignWorkspace', workspaceId, windowId: currentWindowId }`
3. Background saves current window tabs into that workspace, sets `windowMap[windowId] = workspaceId`
4. Popup re-renders — unassigned banner disappears, workspace shows as active
5. No confirmation dialog — "Assign" is clearly labeled and reversible

### Clicking the current window's active workspace

- No-op (existing behavior, `if (!isActive) onSwitch(ws.id)` guard unchanged)
- In-use items owned by this window also no-op (they ARE the active item)

### Window focus changes

- Badge updates via `windows.onFocusChanged` → `updateBadge(workspace, windowId)` in background
- Popup does not need to listen — it re-renders on open

## States Reference

| State | Trigger | Visual |
|-------|---------|--------|
| Active (this window) | `windowMap[currentWindowId] === ws.id` | `#2a2a42` bg + `var(--ws-color)` left border + tab count shows "· active" |
| In-use (other window) | `windowMap[otherWindowId] === ws.id` and `otherWindowId !== currentWindowId` | No dim, no bg change, in-use icon (`#6c7086`) visible at right of name |
| Available | Not in any window's map entry | Default list row, no indicator |
| Unassigned window | `windowMap[currentWindowId]` is null or absent | Banner shows above list; "Assign" buttons visible on all available workspace rows |
| Switching in progress | `isSwitching: true` | All list items `opacity: 0.5` (existing pattern) |
