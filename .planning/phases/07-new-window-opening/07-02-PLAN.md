<objective>
Remove the unassigned-window banner and "Assign Here" buttons from the popup, add a context-sensitive subtitle, and rewire click handlers to support new-window opening via regular click (from unassigned windows), Ctrl+click, and middle-click.

Purpose: This is the user-facing half of Phase 7. With the backend plumbing from Plan 01 in place, this plan transforms the popup so that (a) unassigned windows become neutral launchers that open workspaces in new windows instead of assigning, (b) Ctrl+click and middle-click from any window open in a new window, and (c) a subtitle provides discoverability for the Ctrl+click modifier. The deprecated banner, "Assign Here" buttons, and associated CSS are removed per WIN-02.

Output: Updated popup.html, popup.js, and popup.css implementing all four WIN requirements and all D-01 through D-15 decisions.
</objective>

  <action>
    **In `src/popup/popup.js`**, add the `onOpenInNewWindow` function and rewire the click handlers:

    1. **Add `onOpenInNewWindow` function** in the Actions section (after the existing `onFocusWindow` function):
       ```javascript
       async function onOpenInNewWindow(workspaceId) {
         await browser.runtime.sendMessage({ action: 'openWorkspaceInNewWindow', workspaceId })
         window.close()
       }
       ```
       The function sends the message and closes the popup. No visual feedback is needed (D-14). The response from the background is intentionally not checked — if it fails, the error is logged in the background script. The popup closes naturally regardless.

    2. **Replace the existing click handler** on each workspace `<li>` element. The current handler (inside the `workspaces.forEach` loop) is:
       ```javascript
       li.addEventListener('click', (e) => {
         if (e.target.closest('.ws-actions')) return;
         if (isInUse) {
           onFocusWindow(owningWindowId);
         } else if (!isActive) {
           onSwitch(ws.id);
         }
       });
       ```

       Replace it with this expanded handler that detects Ctrl+click and handles unassigned windows:
       ```javascript
       li.addEventListener('click', (e) => {
         // Don't act if clicking an action button
         if (e.target.closest('.ws-actions')) return

         // D-12: Ctrl+click opens in new window (from any window state)
         if (e.ctrlKey) {
           e.preventDefault()
           if (!isActive) onOpenInNewWindow(ws.id)  // D-13: ignore if active here
           return
         }

         // D-01: If workspace is active in another window, focus that window
         if (isInUse) {
           onFocusWindow(owningWindowId)
         } else if (activeWorkspaceId === null) {
           // D-09: Unassigned window — open in new window (WIN-01)
           if (!isActive) onOpenInNewWindow(ws.id)
         } else if (!isActive) {
           // D-10: Assigned window — switch in current window (existing behavior)
           onSwitch(ws.id)
         }
       })
       ```

       Key differences from the old handler:
       - `e.ctrlKey` check at the top, before any other logic, with `e.preventDefault()` to suppress macOS context menu (Pitfall 4)
       - `activeWorkspaceId === null` branch calls `onOpenInNewWindow` instead of the removed `onSwitch` for unassigned windows (D-09)
       - `isActive` guard on the unassigned-window branch (D-13 — clicking the active workspace from unassigned window does nothing; this case is theoretically impossible since unassigned windows have activeWorkspaceId === null meaning no workspace is "active", but the guard is defensive)
       - Semicolons removed to match codebase convention (ESLint rule)

    3. **Add `auxclick` listener** for middle-click on each `<li>`, immediately after the click listener:
       ```javascript
       li.addEventListener('auxclick', (e) => {
         if (e.button !== 1) return  // only middle-click
         if (e.target.closest('.ws-actions')) return
         e.preventDefault()  // suppress autoscroll (Windows) / paste (Linux/macOS)
         if (!isActive) onOpenInNewWindow(ws.id)  // D-13: ignore if active here
       })
       ```

       This goes right after the `li.addEventListener('click', ...)` block and before the `editBtn.addEventListener('click', ...)` line.
  </action>
