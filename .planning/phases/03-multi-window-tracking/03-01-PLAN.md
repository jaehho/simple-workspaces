<objective>
Replace global activeWorkspaceId with per-window workspace tracking in the background layer.

Purpose: This is the foundation for multi-window support. Every background module must become window-aware before the popup can render per-window state. The global `activeWorkspaceId` is replaced at runtime by a `windowId -> workspaceId` map in `storage.session`, while `storage.local` schema stays unchanged for Phase 4 compatibility.

Output: All four background modules updated -- state.js has window map helpers, workspaces.js functions accept windowId, messaging.js routes new actions, index.js wires window lifecycle and passes windowId through tab events.
</objective>

      // ── Window Event Listeners ────────────────────────────────
      browser.windows.onRemoved.addListener(async (windowId) => {
        await removeWindowEntry(windowId)
      })

      browser.windows.onFocusChanged.addListener(async (windowId) => {
        if (windowId === browser.windows.WINDOW_ID_NONE) return
        const windowMap = await getWindowMap()
        const workspaceId = windowMap[String(windowId)]
        if (workspaceId) {
          const raw = await browser.storage.local.get(['workspaces'])

          const ws = workspaces.find(w => w.id === workspaceId)

          updateBadge(null, windowId)
        }
