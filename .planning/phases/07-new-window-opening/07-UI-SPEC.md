## Focal Point

Primary visual anchor: workspace list items (`.ws-item` rows) — the main interactive surface users scan and click.

The subtitle (`.ws-subtitle`) is secondary supporting text that provides a keyboard hint. It must not compete visually with workspace names: 10px, muted color `#6c7086`, weight 400.

## Interaction Contract

### Click Routing on Workspace `<li>` Items

| Trigger | Condition | Action | Visual Feedback |
|---------|-----------|--------|-----------------|
| `click` (no modifier) | `activeWorkspaceId === null` (unassigned window) | `onOpenInNewWindow(ws.id)` — open in new window | Popup closes naturally; new window appears |
| `click` (no modifier) | `isInUse === true` (active in another window) | `onFocusWindow(owningWindowId)` | Popup closes; owning window focuses |
| `click` (no modifier) | `isActive === true` (active in current window) | No-op | None |
| `click` (no modifier) | Assigned window, not active, not in-use | `onSwitch(ws.id)` — switch in current window | Items fade to 0.5 opacity (existing pattern) |
| `click` with `e.ctrlKey === true` | `isActive === false` | `onOpenInNewWindow(ws.id)` — open in new window | Popup closes; new window appears (D-12) |
| `click` with `e.ctrlKey === true` | `isActive === true` | No-op (D-13) | None |
| `auxclick` with `e.button === 1` | `isActive === false` | `onOpenInNewWindow(ws.id)` — open in new window | Popup closes; new window appears (D-11) |
| `auxclick` with `e.button === 1` | `isActive === true` | No-op (D-13) | None |

The `auxclick` handler must call `e.preventDefault()` before any logic to suppress middle-click autoscroll (Windows) and paste (Linux/macOS).

Ctrl+click handler must call `e.preventDefault()` to suppress context menu on macOS.

Source: CONTEXT.md D-09 through D-13; RESEARCH.md Pattern 3.

### No Extra Feedback for New-Window Open (D-14)

Do not add a loading state, spinner, or toast for new-window open. The popup closes naturally when the new window opens. The existing `onSwitch` opacity-fade is not applied to the new-window path.

### "In-Use" Indicator Remains Unchanged

The `ws-in-use-icon` (two-window SVG, 12×12) and `workspace-item--in-use` class continue to appear when a workspace is active in another window. Clicking an in-use item still focuses the owning window (unchanged from Phase 3). Modifier clicks on in-use items follow the same `isActive` guard (already enforced by `isInUse === true` implying `isActive === false` in the current window).
