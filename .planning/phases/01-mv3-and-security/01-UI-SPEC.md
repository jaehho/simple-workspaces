## SEC-02 SVG Construction Contract

SEC-02 is the only UI-touching change in this phase. The rendered output must be visually
identical before and after.

### Current state (to be replaced)

Two `innerHTML` assignments in `src/popup/popup.js`:

- Line 64: `editBtn.innerHTML = '<svg ...pencil icon...'`
- Line 69: `deleteBtn.innerHTML = '<svg ...cross icon...'`

### Required replacement approach

Use `document.createElementNS` with the SVG namespace. No `innerHTML` anywhere in the codebase
after this change.

**SVG namespace URI:** `http://www.w3.org/2000/svg`

**Pencil icon (edit button) — DOM construction target:**

```
svg: width=14, height=14, viewBox="0 0 16 16", fill="none"
  path: d="M11.5 2.5l2 2-8 8H3.5v-2l8-8z"
        stroke="currentColor", stroke-width="1.3"
        stroke-linecap="round", stroke-linejoin="round"
```

**Cross icon (delete button) — DOM construction target:**

```
svg: width=14, height=14, viewBox="0 0 16 16", fill="none"
  path: d="M4 4l8 8M12 4l-8 8"
        stroke="currentColor", stroke-width="1.3"
        stroke-linecap="round"
```

**SVG attribute namespace:** SVG presentation attributes (`stroke`, `stroke-width`,
`stroke-linecap`, `stroke-linejoin`, `fill`) must be set with `setAttribute` (no namespace
prefix required). Dimensional attributes (`width`, `height`, `viewBox`) likewise use
`setAttribute`.

**Append target:** `editBtn.appendChild(svgEl)` and `deleteBtn.appendChild(svgEl)`.

**Visual contract:** After replacement, edit and delete icon buttons must render with identical
pixel dimensions, stroke appearance, and color behavior (`currentColor` inherits from parent
button's CSS `color` property) as the current `innerHTML` implementation.
